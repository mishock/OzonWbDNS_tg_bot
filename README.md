# OzonWbDNS_tg_bot / AKEM-bot

Telegram-бот **AKEM-bot** (`@OzonWbDNS_bot`) на `aiogram 3`: агрегатор по категориям с выдачей ссылок на поиск Ozon, Wildberries и DNS. Проект приведён в соответствие с файлом ТЗ:

`индивидуальный проект тг бот (1).docx` в корне репозитория.

## Что это за программа

Это учебный Telegram-бот-каталог:
- показывает пользователю категории товаров;
- подбирает товары из трех источников (Ozon, Wildberries, DNS);
- выдает товары порциями по 3 (кнопка «Показать ещё», до 15);
- может работать с AI-подсказками и AI-подборкой (если включен AI-режим).

Бот удобен как шаблон для обучения: здесь видно базовую архитектуру aiogram-приложения, работу с данными и UI-кнопками.

## Полный функционал бота (реестр возможностей)

Ниже перечислено всё, что реализовано в текущей версии кода (`main.py`, `app.py`, `config.py`).

### Команды

| Команда | Действие |
|--------|----------|
| `/start` | Приветствие, краткое описание возможностей, показ главной inline-клавиатуры («Выбрать категорию», переключатель AI). |
| `/help` | Текстовая справка: команды, назначение каждой кнопки, пояснение про ссылки на поиск маркетплейсов; внизу снова главная клавиатура. |

### Главное меню (после `/start` и в ряде сценариев)

- **«Выбрать категорию»** — открывает список категорий из `data/categories.json`.
- **«AI: ВКЛ» / «AI: ВЫКЛ»** — переключает AI **в рантайме** (до перезапуска контейнера): влияет на запросы к DeepSeek и на показ AI-рекомендаций. Начальное значение берётся из `AI_ENABLED` в `.env`.

### Меню категорий

- Кнопки категорий в **две колонки**; у каждой категории тематический emoji из словаря `CATEGORY_ICONS` в `app.py` (при отсутствии slug — иконка 🛍).
- **«Новый поиск»** — сброс к главному экрану (тот же смысл, что старт поиска заново).
- Снова доступен переключатель **AI: ВКЛ/ВЫКЛ**.

### Выдача товаров по категории

1. При выборе категории показывается уведомление callback с «анимацией» ожидания (символы ◐◓◑◒).
2. Формируется список товаров:
   - при **включённом** AI и наличии `DEEPSEEK_API_KEY` — сначала запрос **живого списка** через DeepSeek (JSON-товары); при ошибке или пустом ответе — **fallback** на `data/mock_products.json`;
   - при **выключенном** AI — только каталог из **mock JSON**.
3. Товары **перемешиваются по кругу маркетплейсов** (Ozon → Wildberries → DNS → …), чтобы в порциях по 3 чаще попадались **разные площадки**.
4. Общий лимит на категорию: **не более 15** позиций; с каждой площадки в каталоге берётся не больше **`per_marketplace_limit`** (по умолчанию **5**, задаётся в `config.py` → `Settings.per_marketplace_limit`).
5. Пользователю редактируется служебное сообщение: категория, диапазон «показано X–Y из Z»; в тексте используются «искры» ✨💫⭐🌟.
6. **На первой странице** (первые 3 товара), если AI включён и ключ задан, бот запрашивает **краткую AI-рекомендацию** по списку (учёт цены и рейтинга в промпте) и отправляет отдельным сообщением.
7. Каждый товар — отдельное сообщение: по возможности **фото** (`image_url` → URL; при ошибке Telegram — загрузка по HTTP; если не вышло — **заглушка** 1×1 PNG); подпись в **HTML**.

### Карточка одного товара (поля в тексте)

- Строка **«Название»** (с emoji-категории).
- **«Цена»** в формате с пробелами разрядов и символом **₽**.
- **«Рейтинг»** — только если в данных товара поле заполнено.
- **«Площадка»** — Ozon / Wildberries / DNS с цветным emoji маркетплейса.
- **«Ссылка»** — кликабельная HTML-ссылка «Открыть товар»; фактически это **ссылка на поиск** по названию товара на соответствующем сайте (не прямая карточка SKU).

### Кнопки после порции товаров

| Кнопка | Действие |
|--------|----------|
| **«Показать ещё»** | Показ следующих **трёх** товаров той же категории (тот же отфильтрованный список, другой offset). Скрывается, если товары закончились. |
| **«Другие категории»** | Возврат к списку категорий. |
| **«Новый поиск»** | Возврат к главному экрану с приглашением выбрать категорию. |
| **«AI: ВКЛ/ВЫКЛ»** | То же рантайм-переключение AI. |

Если пользователь нажмёт «Показать ещё», когда страниц больше нет, показывается **всплывающее уведомление** «Больше товаров нет».

### Ошибки и граничные случаи

- Категория не найдена — сообщение об ошибке и сброс клавиатуры сообщения.
- Нет товаров по категории — текст с предложением выбрать другую категорию.
- Непредвиденная ошибка при выдаче — общее сообщение «Произошла ошибка…», запись **stack trace** в лог.
- Парсинг ответа AI на «живые» товары: извлекается JSON-массив из текста; невалидные элементы отбрасываются.

### Маркетплейсы и шаблоны поиска (как в ТЗ)

- **Ozon:** `https://www.ozon.ru/search/?text={query}`
- **Wildberries:** `https://www.wildberries.ru/catalog/0/search.aspx?search={query}`
- **DNS:** `https://www.dns-shop.ru/search/?q={query}`  

`query` — URL-кодированное название товара.

### Техническая сторона

- Стек: **Python 3.11**, **aiogram 3**, long **polling**, разметка сообщений **HTML**.
- **Логирование** в stdout: уровень INFO, формат с временем и именем логгера.
- **Docker:** `docker-compose.yml` — сервис `app`, образ из `Dockerfile`, монтирование каталога проекта в `/app` для правок без пересборки, политика перезапуска **`restart: unless-stopped`**.

## Источники данных

Сейчас используется mock-источник `data/mock_products.json`.
При `AI_ENABLED=false` бот полностью работает на локальном JSON-каталоге:
- `data/categories.json` (категории),
- `data/mock_products.json` (товары).
При `AI_ENABLED=true` бот сначала пытается получить "живую" подборку товаров через AI API, а при неудаче автоматически использует локальный JSON-каталог.

Где менять на реальный API/парсинг:
- `app.py` (класс `MarketplaceService` и инициализация `CatalogService`)

Основная логика бота и формат ответа при этом останутся без изменений.

## Как работать с проектом

1. Заполните переменные в `.env` (см. разделы ниже).
2. Запустите проект в Docker:
   - `docker compose up -d --build`
3. Проверьте, что контейнер жив:
   - `docker compose ps`
4. Проверьте логи бота:
   - `docker compose logs -f app`
5. Остановить проект:
   - `docker compose down`

Типовой цикл при изменениях:
- правите `app.py` или файлы в `data/`,
- перезапускаете сервис `docker compose restart app`,
- проверяете поведение в Telegram.

## Как пользоваться ботом (для пользователя)

1. В Telegram отправьте боту команду `/start`.
2. Нажмите `Выбрать категорию`.
3. Выберите нужную категорию товаров.

   - чтобы сменить категорию — «Другие категории»; начать сначала — «Новый поиск».

## Что за JSON-файлы в `data/`

### `data/categories.json`

Справочник категорий для меню выбора.  
Каждый объект содержит:
- `slug` — технический идентификатор категории (используется в коде и `mock_products.json`);
- `title` — название категории, которое видит пользователь в боте.

Важно:
- `slug` должен быть уникальным;
- если добавляете новую категорию, используйте тот же `slug` в товарах в `mock_products.json`.

### `data/mock_products.json`

Тестовая база товаров для MVP (вместо реального API).

Поля товара:
- `title` — название товара;
- `price` — цена (число);
- `category` — `slug` категории из `categories.json`;
- `marketplace` — одно из: `Ozon`, `Wildberries`, `DNS`;
- `url` — исходная ссылка (в текущей логике заменяется на поисковую ссылку по названию);
- `image_url` — ссылка на картинку или `null`;
- `rating` — рейтинг (опционально);
- `description` — описание (опционально).

## Как настраивать и пополнять товары

### 1) Добавить новую категорию

Файл: `data/categories.json`  
Добавьте объект в массив:

```json
{"slug": "tablets", "title": "Планшеты"}
```

Правила:
- `slug` — латиницей, короткий и уникальный;
- `title` — как увидит пользователь в меню.

### 1.1) Назначить иконку новой категории

Сделайте строго так:

1. Откройте файл `app.py`.
2. Найдите блок `CATEGORY_ICONS = { ... }`.
3. Добавьте строку со своим `slug` и emoji.
4. Сохраните файл и перезапустите бот: `docker compose restart app`.

Пример (добавляем категорию `tablets`):

```python
CATEGORY_ICONS = {
    "smartphones": "📱",
    "laptops": "💻",
    "tvs": "📺",
    "fridges": "🧊",
    "headphones": "🎧",
    "vacuums": "🧹",
    "monitors": "🖥",
    "washers": "🧺",
    "tablets": "📱",
}
```

Что важно:
- ключ должен точно совпадать со `slug` из `data/categories.json`;
- если иконку не добавить, бот использует иконку по умолчанию `🛍`.

### 2) Добавить товары в каталог

Файл: `data/mock_products.json`  
Добавьте новые объекты товаров в массив, например:

```json
{
  "title": "Samsung Galaxy Tab S9",
  "price": 64990,
  "category": "tablets",
  "marketplace": "Ozon",
  "url": "https://www.ozon.ru/product/example",
  "image_url": null,
  "rating": 4.8,
  "description": "Планшет Samsung"
}
```

Что важно проверить:
- `category` должна совпадать со `slug` из `categories.json`;
- `marketplace` только `Ozon`, `Wildberries` или `DNS`;
- `price` — число;
- чем больше данных по всем 3 площадкам, тем лучше работает выдача “по разным источникам”.

### 3) Применить изменения

После правок JSON перезапустите сервис:

```bash
docker compose restart app
```

Потом в Telegram выполните `/start` и проверьте новую категорию/товары.

## Переменные окружения

- `BOT_TOKEN` - основной токен Telegram-бота.
- `TELEGRAM_BOT_TOKEN` - fallback-токен для обратной совместимости.
- `APP_ENV` - окружение запуска (`dev`/`prod`).
- `AI_ENABLED` - включить/выключить AI-рекомендации (`true`/`false`).
- `DEEPSEEK_API_KEY` - ключ DeepSeek API.
- `DEEPSEEK_BASE_URL` - базовый URL DeepSeek API.
- `DEEPSEEK_MODEL` - модель DeepSeek.

## Как отключить AI

- Установите в `.env`: `AI_ENABLED=false`
- Перезапустите контейнер: `docker compose up -d --build`

При `AI_ENABLED=false` бот продолжает работать как обычный MVP без AI.
При `AI_ENABLED=true` AI используется и для рекомендаций, и для попытки получить "живой" список товаров.
Также AI можно включать/выключать прямо в меню бота кнопкой `AI: ВКЛ/ВЫКЛ` (переключение действует до перезапуска контейнера).

## Ограничение выдачи

- Лимит в коде: до 5 товаров на маркетплейс (`per_marketplace_limit=5` в `config.py`), суммарно не более 15 товаров на категорию.
- За один показ бот отправляет 3 товара.
- В каждом блоке из 3 бот старается показать товары из разных источников: Ozon + Wildberries + DNS.
- По кнопке «Показать ещё» бот добавляет следующие 3 товара, пока не дойдёт до конца списка (максимум 15).
